🎥 Chromecast Receiver Test Page

Quick Tests

Pre-Deployment Checklist

  • Receiver HTML file exists and is valid
  • HTTPS hosting is available (required for Chromecast)
  • WebRTC is supported in the receiver environment
  • Cast SDK is properly included
  • Custom namespace matches sender: urn:x-cast:com.chromecast.screenmirror

Deployment URLs

GitHub Pages:

https://yourusername.github.io/reponame/index.html

Firebase Hosting:

firebase deploy
# URL will be shown after deployment

Custom Server:

https://your-domain.com/receiver/index.html

Register Your App

  1. Go to Google Cast Console
  2. Click "Add New Application"
  3. Select "Custom Receiver"
  4. Enter your receiver URL (must be HTTPS)
  5. Save and copy your Application ID
  6. Set it in your C# app: 
    set CHROMECAST_APP_ID=YourAppId

Testing After Deployment

  1. Deploy your receiver to HTTPS hosting
  2. Register the app in Cast Console
  3. Wait ~15 minutes for Cast to propagate
  4. Run your C# application with the new App ID
  5. Use Chrome Cast Debugger to see receiver logs: 
    chrome://inspect/#devices

Common Issues

  • Receiver not loading: Verify HTTPS and correct URL in Cast Console
  • WebRTC fails: Check ICE servers and network firewall
  • No video: Verify namespace matches and check browser console
  • Connection timeout: Ensure sender and Chromecast are on same network
